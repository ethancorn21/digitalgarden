# C(sharp) Interfaces and Dynamic Routing Protocols

C# interfaces are contracts. The caller does not need to know how a method works, only that it exists inside a class.

This abstraction is similar to routers and dynamic routing protocols. A router does not need to know how OSPF calculates cost, only that it is there and returns a value.
